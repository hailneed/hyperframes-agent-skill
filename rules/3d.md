## Approach 1 — CSS 3D transforms (lightweight)

No library needed. Use `perspective` + `rotateX/Y/Z` for card flips, perspective reveals, and depth effects.

```html
<div id="card-3d"
     class="clip"
     data-start="1"
     data-duration="6"
     data-track-index="2"
     style="position: absolute; top: 290px; left: 560px;
            width: 800px; height: 500px;
            transform-style: preserve-3d; perspective: 1200px;">
  <div id="card-inner" style="width:100%; height:100%; position:relative; transform-style:preserve-3d;">
    <div id="card-front"
         style="width:100%; height:100%; position:absolute; backface-visibility:hidden;
                background:#111; border-radius:24px; display:flex; align-items:center;
                justify-content:center; color:#fff; font-size:48px;">
      Front
    </div>
    <div id="card-back"
         style="width:100%; height:100%; position:absolute; backface-visibility:hidden;
                background:#0066ff; border-radius:24px; display:flex; align-items:center;
                justify-content:center; color:#fff; font-size:48px;
                transform:rotateY(180deg);">
      Back
    </div>
  </div>
</div>

<script>
  window.__timelines = window.__timelines || {};
  const tl = gsap.timeline({ paused: true });
  tl.to("#card-inner", { rotationY: 180, duration: 1.2, ease: "power2.inOut" });
  window.__timelines["card-3d"] = tl;
</script>
```

## Approach 2 — Three.js (full WebGL)

Load Three.js from CDN and render into a `<canvas>` clip. The Three.js animation loop must be driven by GSAP, not `requestAnimationFrame`.

```html
<canvas id="three-canvas"
        class="clip"
        data-start="0"
        data-duration="10"
        data-track-index="2"
        style="position: absolute; top: 0; left: 0; width: 1920px; height: 1080px;">
</canvas>

<script src="https://cdn.jsdelivr.net/npm/three@0.166/build/three.min.js"></script>
<script>
  window.__timelines = window.__timelines || {};

  // Setup scene
  const canvas = document.getElementById('three-canvas');
  const renderer = new THREE.WebGLRenderer({ canvas, antialias: true, alpha: true });
  renderer.setSize(1920, 1080);

  const scene = new THREE.Scene();
  const camera = new THREE.PerspectiveCamera(60, 1920 / 1080, 0.1, 1000);
  camera.position.z = 5;

  const geometry = new THREE.BoxGeometry(2, 2, 2);
  const material = new THREE.MeshStandardMaterial({ color: 0x0066ff, metalness: 0.5 });
  const cube = new THREE.Mesh(geometry, material);
  scene.add(cube);

  const light = new THREE.DirectionalLight(0xffffff, 2);
  light.position.set(5, 5, 5);
  scene.add(light);
  scene.add(new THREE.AmbientLight(0xffffff, 0.4));

  // Drive rotation via GSAP — NOT requestAnimationFrame
  const proxy = { rotation: 0 };
  const tl = gsap.timeline({ paused: true });
  tl.to(proxy, {
    rotation: Math.PI * 2,
    duration: 10,
    ease: "none",
    onUpdate() {
      cube.rotation.y = proxy.rotation;
      renderer.render(scene, camera);
    },
  });

  window.__timelines["three-canvas"] = tl;
</script>
```

**Critical:** Never use `requestAnimationFrame` for Three.js in HyperFrames. The renderer must be driven by GSAP `onUpdate` so frames are captured deterministically.

## Approach 3 — CSS perspective text reveal

A dramatic 3D text reveal with no library dependency:

```html
<div id="reveal-stage"
     class="clip"
     data-start="1"
     data-duration="5"
     data-track-index="3"
     style="position: absolute; top: 380px; width: 1920px; text-align: center;
            perspective: 1400px; perspective-origin: 50% 50%;">
  <div id="reveal-text"
       style="color: #fff; font-size: 96px; font-weight: 900;
              font-family: Inter, sans-serif; transform: rotateX(90deg); opacity: 0;">
    GAME CHANGER
  </div>
</div>

<script>
  window.__timelines = window.__timelines || {};
  const tl = gsap.timeline({ paused: true });
  tl.to("#reveal-text", {
    rotationX: 0, opacity: 1,
    duration: 0.8,
    ease: "power3.out",
  });
  window.__timelines["reveal-stage"] = tl;
</script>
```

## Catalog 3D blocks

HyperFrames ships pre-built 3D effects:

```bash
npx hyperframes add perspective-reveal
npx hyperframes add flip-transition
```

Prefer catalog blocks for common 3D transitions — they are pre-tested and render correctly.
